# Games Automation

Before I learnt about Tas or memory manipulation, I tried to automate some games gameplay using pyautogui. using a cycle of press-button -> time.sleep() -> repeat. it was fun to do at the time, but it is pretty useless now because it uses hardcoded coordinates and it only worked on my old computer that I don't have anymore. the pokemon one is the best because it reads the screen to decide what to do, and it contains a database for game state.

## Projects

- [Sonic](./sonic_automation) : a script to automate the gameplay of Sonic the Hedgehog. it is incomplete, it only open the emulator iirc.
- [Mario](./mario) : a script to automate the gameplay of Super Mario Bros. it is incomplete, it only open the emulator iirc.
- [Pokemon](./pokemon) : a script to automate the gameplay of Pokemon Emerald. I think I got to the first trainer battle, but I don't remember for sure. it is the most complete of the three, it reads the screen to decide what to do, and it contains a database for game state. it is pretty impressive for a script that I did when I was 18 years old. but it doesn't work anymore 😭😢😢
